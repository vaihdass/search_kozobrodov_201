## Задание 2. Токенизация и лемматизация

Из скачанных HTML-страниц извлекается текст рецептов, токенизируется и группируется по леммам. Для каждой страницы создаётся отдельный файл токенов и файл лемм.

### Структура проекта

```
hw2/
    tokenizer.py    # скрипт токенизации и лемматизации
    tokens/         # результат: файл токенов на каждую страницу (179 файлов)
    lemmas/         # результат: файл лемм на каждую страницу (179 файлов)
```

### Пример результата

tokens/0001.txt:
```
абрикос
абрикосам
абрикосами
```

lemmas/0001.txt:
```
абрикос абрикос абрикосам абрикосами абрикосов абрикосы
абсолютно абсолютно
авокадо авокадо
```

### Зависимости

```bash
pip install beautifulsoup4 lxml pymorphy3 nltk
```

### Запуск

```bash
cd hw2
python3 tokenizer.py
```

Скрипт прочитает HTML из `../pages/`, создаст директории `tokens/` и `lemmas/` с файлами по одному на каждую страницу.

## Задание 4. TF-IDF

Для каждого документа считаем TF-IDF по токенам и по леммам.

### Формулы

Для токенов (конкретных форм слова):
- TF = кол-во вхождений токена в документ / общее кол-во токенов в документе
- IDF = log(кол-во документов / кол-во документов, содержащих этот токен)

Для лемм (нормальных форм):
- TF = суммарное кол-во вхождений всех словоформ леммы / общее кол-во токенов в документе
- IDF = log(кол-во документов / кол-во документов, содержащих хотя бы одну словоформу леммы)

TF-IDF = TF * IDF (чем выше, тем слово важнее для документа)

### Структура проекта

```
hw4/
    tfidf.py        # подсчет TF-IDF
    tokens/         # результат по токенам (179 файлов)
    lemmas/         # результат по леммам (179 файлов)
```

Формат выходных файлов: `<термин> <idf> <tf-idf>`

### Пример

tokens/0001.txt:
```
большого 4.494239 0.011985
вашей 2.789491 0.007439
взгляд 2.622436 0.006993
```

lemmas/0001.txt:
```
большой 1.549800 0.004133
ваш 1.275363 0.003401
взгляд 2.622436 0.006993
```

### Зависимости

```bash
pip install beautifulsoup4 lxml pymorphy3 nltk
```

### Запуск

```bash
cd hw4
python3 tfidf.py
```

## Задание 5. Векторный поиск (CLI)

Принимает текстовый запрос на естественном языке и находит наиболее релевантные документы, используя косинусное сходство TF-IDF векторов.

### Требования

- [Go](https://go.dev/dl/) 1.21+
- Python 3.8+ (для одноразовой генерации данных)
- Python-зависимости: `pip install beautifulsoup4 lxml pymorphy3 nltk`

### Как работает

Подготовка данных (Python, один раз):
1. gen_data.py читает TF-IDF файлы из hw4/lemmas/ и HTML страницы из pages/
2. Формирует JSON-файлы: TF-IDF вектора документов, IDF значения лемм и метаданные документов

Поиск (Go):
1. При запуске загружает JSON-данные в память, предвычисляет нормы документов и лемматизирует все токены всех документов (один раз)
2. Если морфологический словарь SteosMorphy отсутствует - автоматически собирает его из Go module cache
3. Запрос токенизируется (выделяются русские слова), каждое слово приводится к лемме через SteosMorphy
4. Строится TF-IDF вектор запроса (с теми же IDF весами, что и в индексе)
5. Для каждого документа считается косинусное сходство с вектором запроса
6. Результаты сортируются по убыванию сходства, для топ-N генерируется сниппет

Выбор сниппета:
- Текст документа токенизируется и лемматизируется при загрузке (предвычислено)
- Из совпавших с запросом позиций выбирается окно 200 символов с максимальным числом различных лемм запроса
- Слова из сниппета, совпавшие с запросом, возвращаются для подсветки

### Структура проекта

```
hw5/
    cmd/
        cli/main.go                    # точка входа CLI
        api/main.go                    # точка входа HTTP сервера
    internal/
        search/
            index.go                   # загрузка данных, инициализация движка
            scorer.go                  # косинусное сходство, ранжирование
            snippet.go                 # выбор сниппета, предвычисление токенов
            lemmatize.go               # лемматизация через SteosMorphy с кешем
        service/
            search.go                  # сервисный слой, Result DTO
        transport/
            cli/runner.go              # вывод результатов в терминал
            web/handler.go             # HTTP handler + шаблон
    gen_data.py                        # одноразовая генерация JSON из hw4
    data/
        index.json                     # TF-IDF вектора документов
        idf.json                       # IDF значения лемм
        docs.json                      # заголовки, тексты и URL документов
```

### Запуск

```bash
cd hw5
python3 gen_data.py                  # генерация JSON (один раз)
go run ./cmd/cli                     # интерактивный поиск (по умолчанию)
go run ./cmd/cli -i "ваш запрос"     # одноразовый поиск
```

При первом запуске Go программа автоматически соберет морфологический словарь SteosMorphy (~445 МБ) из Go module cache. Файл `data/morph.dawg` создается один раз и не коммитится в git.

## Поисковая система (веб-интерфейс для задания 5)

Поверх векторного поиска из задания 5 реализован HTTP API с веб-интерфейсом.

### Требования

- [Go](https://go.dev/dl/) 1.21+
- Python 3.8+ и зависимости из задания 5 (для генерации данных)
- make (опционально)

### Как работает

- GET `/api/search?q=запрос&limit=10` - возвращает JSON с результатами (limit: 10/15/30 и 0 для всех результатов)
- Для каждого результата возвращается заголовок, URL, сниппет и список слов для подсветки
- Веб-интерфейс отображает результаты с подсветкой совпавших слов, ссылками на оригинальные страницы и выбором количества результатов

### Структура проекта

```
hw5/
    cmd/api/main.go                    # точка входа HTTP сервера
    internal/transport/web/handler.go  # HTTP handler
    templates/index.html               # веб-интерфейс
```

### Запуск

```bash
cd hw5
python3 gen_data.py   # генерация JSON (один раз, если не сделано)
make run              # запуск на http://localhost:8080
```

Если `make` недоступен:

```bash
go run ./cmd/api/
```
